## 🧰 Troubleshooting & Common Issues

Even well-built scripts encounter edge cases. Here are the most reported problems and their solutions:

### ❌ GUI doesn't appear after execution

**Cause:** Injection happened before the game fully loaded
**Fix:** Wait for the full game world to render (loading bar at 100%), then re-inject. If the problem persists, try switching to a less populated server.

### ❌ Auto Farm stops after a few minutes

**Cause:** Roblox's 20-minute idle disconnect triggered despite AntiAFK
**Fix:** Ensure `AntiAFK = true` is set in your CONFIG block. Some executors need special flags to allow heartbeat hooks — check your executor's documentation for "Heartbeat permission."

### ❌ ESP labels are misaligned or floating off-screen

**Cause:** Screen resolution mismatch or monitor scaling issue
**Fix:** Set your display scaling to 100% in Windows display settings. For macOS, set resolution to "More Space" mode in System Settings → Displays.

### ❌ Script crashes on Android

**Cause:** Insufficient RAM or background app conflict
**Fix:** Close all background apps before running. Restart your device, open only Roblox and the executor, then inject.

### ❌ Speed Multiplier causes rubber-banding

**Cause:** Value set too high relative to server tolerance
**Fix:** Reduce `SpeedMultiplier` to between 1.5 and 2.0. Values above 3.0 consistently trigger server-side corrections.

### ❌ "Attempt to index nil value" error in output

**Cause:** Game update changed internal service names
**Fix:** Check the repository for a patch update. Enable `AutoUpdate = true` in CONFIG so future updates are pulled automatically without manual re-download.

### ❌ Teleport sends character to wrong location

**Cause:** Map layout changed in a recent Horse Life update
**Fix:** Disable teleport presets and use manual ESP tracking instead until a patch is deployed. Report the issue on the GitHub Issues tab.
